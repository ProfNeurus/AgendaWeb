# GUÍA DE ACTUALIZACIÓN Y VERSIONADO (GIT + GITHUB)

Este archivo contiene los comandos y pasos necesarios para mantener tu proyecto 'AgendaWeb' respaldado en GitHub y actualizado.

## 1. FLUJO DE TRABAJO DIARIO (Local -> GitHub)
Cada vez que realices cambios en tu código y quieras guardarlos:

1. Ver el estado de los cambios:
   git status

2. Preparar los archivos para el envío:
   git add .

3. Crear un punto de restauración (Commit):
   git commit -m "Descripción breve de los cambios"

4. Subir a GitHub:
   git push origin main

---

## 2. ACTUALIZACIÓN EN EL SERVIDOR (GitHub -> Ubuntu)
Si el proceso es manual, debes entrar a tu servidor Ubuntu y ejecutar:

1. Entrar a la carpeta:
   cd /ruta/de/tu/proyecto

2. Descargar los cambios:
   git pull origin main

3. Reiniciar el proceso (si usas PM2):
   pm2 restart agenda-web

---

## 3. CONFIGURACIÓN INICIAL DE IDENTIDAD
Si es la primera vez en una máquina nueva, configura tu nombre y correo:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

---

## 4. GESTIÓN DE VERSIONES (TAGS)
Para marcar una versión estable (ejemplo: v1.0, v1.1):

1. Crear etiqueta de versión:
   git tag -a v1.0 -m "Descripción de la versión"

2. Subir la etiqueta a GitHub:
   git push origin v1.0

3. Borrar una etiqueta (si te equivocas):
   git tag -d v1.0           (Local)
   git push origin --delete v1.0 (GitHub)

---

## 5. RECURSOS ÚTILES
Repositorio actual: https://github.com/ProfNeurus/AgendaWeb
Branch principal: main


Trajaar en distintas VERSIONES

1. Etiquetar la versión actual (Tag)
Una "etiqueta" o Tag es como una foto fija de tu código en este momento. Como ya tienes las versiones v1.0 y v1.1, te sugiero llamar a esta v1.2.

Ejecuta este comando para marcar el estado actual:

powershell
git tag -a v1.2 -m "Version definitiva con auto-refresco cada 15 min"
2. Crear una nueva rama para la Versión 2.0 (Branch)
En lugar de trabajar sobre la misma línea, puedes crear una "rama" nueva. Esto separa completamente el desarrollo de la versión 2.0 de la versión actual.

Ejecuta:

powershell
git checkout -b version-2.0
Ahora todo lo que hagas y guardes se quedará en la rama version-2.0.

¿Cómo volver atrás si lo necesitas?
Si en algún momento quieres volver a ver o usar la versión definitiva (v1.2), solo tienes que hacer:

powershell
git checkout v1.2
Y para volver a seguir trabajando en la 2.0:

powershell
git checkout version-2.0
